# AGENTS.md

This file provides guidance to Codex (Codex.ai/code) when working with code in this repository.

## Project Overview

**GeoViewer 3D - Pro GPX Suite & Print Studio** is a zero-build web application for GPX track editing, 3D map viewing, and topographic printing. It uses native ES modules — no bundler, no package manager, no build step. All dependencies are loaded via CDN at runtime.

To run: open `index.html` directly in a browser (or serve locally via `python3 -m http.server` — ES modules require a server or `file://` with CORS disabled).

> **Note:** The project was previously a single monolithic `index.html`. It has been refactored into the modular structure described below. Do not suggest reverting to a single-file approach.

## Tech Stack

All loaded via CDN — nothing to install:
- **MapLibre GL JS** v3.6.2 — WebGL vector map rendering with 3D terrain
- **Turf.js** v6 — geometry operations
- **Chart.js** — elevation and slope profile charts
- **Tailwind CSS** — utility-first styling
- **Lucide Icons** — SVG icon set
- **OSRM** (external API) — snap-to-road routing
- **Nominatim / OpenStreetMap** (external API) — geocoding/search

UI text, comments, and variable names are in **Italian**.

## Architecture

### Shared State (`state.js`)

All application data is exported from `state.js`. The primary structure is:

```javascript
// state.js
export let tracks = [];        // array of track objects
export let map = null;         // MapLibre instance
export let undoStack = [];     // undo history
// ... other globals
```

Modules that need to mutate state import named getters/setters or import the variable directly. **Never redeclare these variables in other modules.**

The core `tracks` data shape:
```javascript
{
  id, name, desc, color, width, visible, waypointsVisible,
  segments: [
    { id, name, points: [{ lat, lon, ele, isUserClicked }, ...] }
  ],
  waypoints: [
    { id, name, lat, lon, ele, symbol, visible }
  ]
}
```

Undo history: before any destructive state mutation, call `saveHistoryState()` (in `tracks.js`) to support Ctrl+Z.

### Two Master Refresh Functions

After any state change, call one or both:
- **`updateMapData()`** (`map.js`) — rebuilds GeoJSON and repaints all MapLibre layers
- **`renderGisTree()`** (`ui.js`) — rebuilds the right-side panel (tracks, segments, waypoints list)

Visibility filtering happens inside `updateMapData()`: hidden layers are omitted from GeoJSON, which immediately reduces GPU load. Stats and elevation charts also exclude hidden layers.

### Dependency Injection Pattern (`ui.js`)

`ui.js` uses an `injectDeps(deps)` function to receive references to functions from other modules at startup. This avoids ES module circular evaluation issues while preserving runtime cross-module calls. `main.js` calls `injectDeps()` before `init()`.

### Window Globals for Inline Handlers

HTML generated by `renderGisTree()` uses `onclick="..."` string attributes. All functions referenced this way are exposed via `window.xxx = xxx` in `main.js`. If you add a new inline handler in `renderGisTree`, remember to expose it in `main.js`.

### Module Responsibilities

| Module | Responsibility |
|---|---|
| `state.js` | Single source of truth for all mutable state |
| `utils.js` | Pure functions with no side effects or imports |
| `map.js` | MapLibre lifecycle, sources, layers, terrain |
| `stats.js` | Chart.js elevation profile, distance/slope math |
| `tracks.js` | Point editing, undo, snap-to-road (OSRM) |
| `waypoints.js` | Waypoint CRUD, modal editor, drag reposition |
| `gpx.js` | GPX XML parsing and serialization |
| `print.js` | A4 print engine, drag grid, page capture |
| `ui.js` | GIS tree panel, toolbar, toast, modal wiring |
| `main.js` | Bootstrap: wires deps, exposes globals, calls init |

## Key Constraints — Do Not Change These

### Print Engine
The print pipeline requires:
1. `preserveDrawingBuffer: true` in the MapLibre initialization (`main.js` / `map.js`) — required for WebGL canvas capture
2. The `idle` event listener — confirms all tiles loaded before each screenshot

Flow: user drags virtual A4 grid → app moves map under each cell → waits for `idle` → captures WebGL buffer → composes pages via `@media print` CSS.

### Performance-Sensitive Code
- **Haversine loops** (`stats.js`): raw math, no object allocation — do not refactor to Turf wrappers
- **Douglas-Peucker** (`gpx.js`): runs on export to reduce GPX file size
- **Large track rendering**: MapLibre WebWorkers handle tiling at render time — do not add `zoomend` re-render listeners
